How GPU communication shapes network topology

A field note on why collective-communication patterns — not raw FLOPs — increasingly decide how AI data-center fabrics are wired, scheduled, and operated.

Fabric Research
updated 2026-05-12 · 11 min read · v2

01Context

Modern model serving rarely fails because a single GPU is slow. It degrades when many GPUs cannot exchange gradients and activations fast enough. As models cross the memory ceiling of one device, the unit of compute becomes the pool, and the fabric between devices becomes the bottleneck.

This note traces how collective patterns like all-reduce and all-to-all push specific topology decisions — rail-optimized spines, sr-iov partitioning, and ECN-based congestion control.

Premise. The network is not plumbing beneath the model — it is part of the model's execution graph. Topology is a scheduling decision.

02System model

We model an inference route as a path across four stages. Each hop has a latency budget; the route is healthy only when every hop stays inside it.

03Design

The scheduler co-locates tensor-parallel ranks on the same rail to keep all-reduce on the high-bandwidth domain, and spreads pipeline stages across rails to overlap communication with compute.

schedule.py python
# place tensor-parallel ranks on one rail; pipeline across rails
def place(pool, tp, pp):
    rails = pool.rails(bandwidth="400G")
    for stage in range(pp):
        rail = rails[stage % len(rails)]
        rail.bind(tp_group=tp, policy="rail-optimized")
    return pool.commit(congestion="ecn")

04Trade-offs

  • Rail-optimized placement cuts all-reduce tail latency, but reduces scheduler freedom under fragmentation.
  • ECN keeps queues shallow at the cost of occasional throughput dips during bursts.
  • Paged kv-cache raises effective batch size but adds indirection on the hot path.

05Failure modes

The most common production incident is not a crash — it is silent backpressure. When the GPU pool saturates, queue depth climbs while latency degrades gradually. The route stays healthy by liveness checks yet misses its SLA. Observability must watch derivatives, not just absolute values.

Watch the slope. Alert on rate-of-change of queue depth and TTFT, not thresholds alone. By the time a threshold trips, the route has already been degrading for minutes.

06Summary

Topology is a first-class part of model serving. Wire the fabric for the collective patterns the model actually runs, schedule with the network in mind, and observe the system the way it actually fails — quietly, at the edges.
